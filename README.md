# GCC Supermarket Sales Analysis

## Overview
This project analyzes supermarket sales data to identify revenue patterns, customer behavior, product performance, and payment trends using Python, SQL, and Power BI.

## Objective
The goal of this project is to explore sales performance and generate business insights that can help improve decision-making in areas such as product strategy, customer segmentation, and revenue tracking.

## Dataset
- Dataset name: Sales of a Supermarket
- Domain: Supermarket Sales
- Source: Kaggle 
- Records: 1000

## Tools Used
- Python: Data cleaning, preprocessing, and exploratory data analysis
- SQL: Querying and extracting business insights
- Power BI: Interactive dashboard creation

## Project Workflow
1. Collected and understood the dataset.
2. Cleaned and prepared the data using Python.
3. Performed exploratory data analysis to detect patterns and trends.
4. Wrote SQL queries to answer business-related questions.
5. Built an interactive Power BI dashboard for visualization and reporting.

## Key Business Questions
- Which city generated the highest revenue?
- Which product line performed best?
- Which payment method was used most?
- How does revenue vary by hour and month?
- How do different customer types compare in revenue contribution?

## Dashboard Highlights
- Total Revenue: AED313.76K
- Total Orders: 991
- Total Quantity Sold: 5K
- Average Order Value: 316.60
- Average Rating: 6.98

## Key Insights
- Abu Dhabi, Sharjah, and Dubai contributed nearly similar revenue levels.
- Food and beverage was among the top-performing product lines.
- Revenue varied noticeably by hour, showing peak sales periods.
- Members contributed slightly more revenue than normal customers.
- Monthly revenue dipped in February and recovered in March.
